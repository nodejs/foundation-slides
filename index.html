The Node.js Foundation

The Node.js Foundation

A public non-profit member supported organization. Created as a neutral home for the Node.js project.

Shortly after its creation io.js and Node.js merged under the foundation and began converged releases.

In November the foundation will begin accepting additional top level projects.

Corporate Members

Foundation Structure

The foundation has two sides, one technical and one administrative. The TSC (Technical Steering Committee) is made of open source contributors. The Board of Directors oversees the administration of the foundation.

Individual Membership

  • In addition to corporate members the foundation also has an Individual Membership Program.
  • Membership is $100 per year, discounted to $25 for students and free to contributors.
  • Members will elect representation to the Board of Directors to represent them as a class.
  • Future benefits may include email addresses, swag and conference discounts.

The Node.js Project

  • The Node.js project consists of over 20 Working Groups.
  • ~400 project members across all working groups.
  • The primary governing body is the TSC but each Working Group has autonomy.

Governance

  • Core and nearly all WGs operate under liberal contribution policies. Committers are added shortly after any notable contribution is landed.
  • Every contribution is a Pull Request. No committer can land their own pull request. All contributions are reviewed by the committers. Most contributions land without involving the TSC.
  • Controversial contributions are escalated to the TSC.
  • The TSC operates on a Consensus Seeking Model. A consensus is usually reached but if one cannot be reached a simple majority wins vote can occur.
  • No more than 1/4 of the TSC can be affiliated with the same company.

Getting Involved: GitHub

  • Almost all communication happens on GitHub. Issue trackers are used in place of mailing lists.
  • All contributions are Pull Requests.
  • Check the README for each project to see how to contribute.

Getting Involved: Evangelism WG

  • Writes weekly project updates for the blog.
  • Responds to requests in the help repo.
  • Promotes community meetups and conferences.

Getting Involved: Website WG

  • Maintains the website tooling which builds the website from templates and markdown and supports localization.
  • Approves new blog posts and content with the participation of other WGs (Evangelism WG & Documentation WG).
  • Constant iterative style and content improvements, contributions welcome :)

Getting Involved: Documentation WG

  • Maintains the documentation section of the website.
  • Writes new tutorials and articles for the upcoming Knowledge Base.
  • Maintains tooling for building the Knowledge Base.

Getting Involved: Hardware WG

  • Exciting work has been done by independent, established JS hardware communities.
  • Hardware WG is a communication conduit between Node.js and those communities.
  • Mission is to reduce barriers to entry for using and developing hardware libraries in Node.js.
